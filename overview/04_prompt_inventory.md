## 4. 已实际出现的触发型 Prompt 形态

除了自然语言指令，当前项目里已经出现并实际使用的触发型 Prompt 主要有：

- `[Log]` / `[log]`
  - 用于记录 Prompt 历史与线程上下文
- `[Commit]` / `[commit]`
  - 用于触发基于 `git diff` 的提交识别与真实 commit
- `[scores]`
  - 用于触发评分标准对齐，而不是默认立即打分

其中：

- `[commit]` 在项目对话中已经被使用多次
- `[log]` 已在 `P35` 中作为真实触发形式出现
- `[scores]` 当前更多以规则和资产形式出现，未在这份清单中形成高频原文记录

## 5. 按意图归类后的 Prompt 主题

如果按任务意图归类，当前项目里已经出现的 Prompt 大致可以分成 10 类：

1. **仓库盘点类**
   - 例如：分析项目现状、理解当前仓库阶段
2. **规则设计类**
   - 例如：改 commit 规则、补 prompt 规则、补评分规则
3. **方案规划类**
   - 例如：如何开始设计、两天内怎么收 scope、是否依赖 API
4. **难点分析类**
   - 例如：项目到底难在哪、传播链和内容核查分析是否够透
5. **文档沉淀类**
   - 例如：补一份文档、补一份复盘、补执行计划
6. **仓库整理类**
   - 例如：整理目录、整理索引、整理 overview
7. **产品定位类**
   - 例如：到底是谣言检测、新闻核查还是新闻观察员工作台
8. **交互与前端形态类**
   - 例如：是否做成聊天式分析卡片、是否展示思考过程
9. **技术 / 资源决策类**
   - 例如：前后端用什么、是否找开源底座、是否依赖 key、是否有免费 API、账号能否复用
10. **触发执行类**
   - 例如：`[commit]`、`[log]`

## 6. 当前最常出现的 Prompt 模式

结合当前线程与历史记录，出现频率最高的 Prompt 模式有：

- `请先分析 / 评估当前情况`
- `请把某个分析沉淀成文档`
- `请根据现有文档继续补规则`
- `请先整理再提交`
- `请先判断有没有开源可借`
- `请判断 API / key / 免费额度 / 账号能否复用`
- `请把结论继续收口成 V1`
- `请直接触发一次 commit 或 log`

这说明当前对话风格比较稳定，主要围绕：

- 先定位问题
- 再验证资源与约束
- 再收口成方案
- 再沉淀文档与规则
- 最后触发执行动作

- 早期未进入 `prompt-history.md` 的完整原文对话
