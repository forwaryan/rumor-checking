本文件用于整理当前项目内**可追溯**的 Prompt 使用记录。

本文件只整理**用户侧 Prompt / 指令**，不包含系统提示词、开发者提示词和助手回复。

## 4. 已出现的触发型 Prompt 形态

除了自然语言指令，当前项目里已经出现并实际使用的触发型 Prompt 主要有：

- `[Log]`
  - 用于记录 Prompt 历史与线程上下文
- `[Commit]` / `[commit]`
  - 用于触发基于 `git diff` 的提交识别与真实 commit
- `[scores]`
  - 用于触发评分标准对齐，而不是默认立即打分

其中：

- `[commit]` 在实际对话中已经被使用多次
- `[Log]` 和 `[scores]` 当前更多以规则与资产形式出现

## 5. 按意图归类后的 Prompt 主题

如果按任务意图归类，当前项目里已经出现的 Prompt 大致可以分成 7 类：

1. **仓库盘点类**
   - 例如：分析项目现状、理解当前仓库阶段
2. **规则设计类**
   - 例如：改 commit 规则、补 prompt 规则、补评分规则
3. **方案规划类**
   - 例如：如何开始设计、两天内怎么收 scope、是否依赖 API
4. **难点分析类**
   - 例如：项目到底难在哪、传播链和内容核查分析是否够透
5. **文档沉淀类**
   - 例如：补一份文档、补一份复盘、补执行计划
6. **仓库整理类**
   - 例如：整理目录、整理索引、整理 overview
7. **触发执行类**
   - 例如：`[commit]`

## 6. 当前最常出现的 Prompt 模式

结合当前线程与历史记录，出现频率最高的 Prompt 模式有：

- `请先分析 / 评估当前情况`
- `请把某个分析沉淀成文档`
- `请根据现有文档继续补规则`
- `请先整理再提交`
- `请直接触发一次 commit`

这说明当前对话风格比较稳定，主要围绕：

- 先分析
- 再沉淀文档
- 再整理结构
- 最后提交

## 7. 当前清单的边界

本文件当前**没有**包含以下内容：

- 早期未进入 `prompt-history.md` 的完整原文对话
- Codex 系统提示词或规则注入内容
- 助手侧生成的内部子任务提示词
- 后续可能新增但尚未入库的 Prompt 资产版本记录

如果后面你希望把这份清单继续升级，可以再拆成：

- `prompt_inventory_user.md`：只保留用户 Prompt
- `prompt_inventory_triggers.md`：只保留 `[Log]`、`[Commit]`、`[scores]` 一类触发指令
- `prompt_inventory_assets.md`：整理真正进入规则、模板和评测的 Prompt 资产
