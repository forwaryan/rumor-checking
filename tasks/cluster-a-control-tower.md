# Cluster-A / Control Tower

## 这个子 task 是干什么的

这个工作包负责整个 V1 项目的主控与收口，不直接承担大块业务开发，而是负责边界冻结、优先级管理、跨线程协调和最终集成判断。

## 为什么要有这个子 task

如果没有一个明确的主控工作包，前端、后端、检索、测试会各自推进，最后很容易出现 schema 漂移、目录混乱、任务顺序错位和集成返工。

## 为什么这个子 task 可以并行

它主要处理的是约束、节奏和集成决策，和前端页面开发、后端主链路开发、检索模块开发不是同一种改动面。只要它不直接去改其他 cluster 的核心实现文件，就可以与其他任务并行推进。

## 窗口执行 Prompt（全局）

```text
你现在负责 Cluster-A / Control Tower。
你的目标是维护当前阶段的总控口径，而不是编写大块业务功能。
请先完整阅读本文件、tasks/README、顶层 README，以及当前正在进行中的 cluster 文件；只围绕本文件中“进行中/未完成”的子任务推进。
执行时必须先把当前要处理的子任务拆成 3 到 7 个更细步骤，并先把“本轮执行任务 / 执行步骤”写回本文件对应子任务下，再开始改文档或状态文件。
你可以修改 tasks/、README、overview/ 等说明文档来同步范围、状态、优先级和交接建议，但不要直接改前后端核心业务实现，除非只是为状态说明补链接或备注。
完成后必须：
1. 回写本文件中对应子任务的状态，并补充本轮完成记录：改了哪些文件、怎么完成、验证如何、剩余问题是什么。
2. 更新需要同步的总控说明文档。
3. 给出下一步应交给哪个 cluster，以及是否达到 go / no-go 条件。
如果用户要求 [log]，同步更新 prompt-history.md。
```

## 当前实现判断

- 当前仓库已经不再是“实现准备阶段”，而是“最小闭环已成形，关键缺口待补齐”。
- `tasks/` 主状态板已经同步到最新一轮实现进度，但后续仍需每轮执行持续回写，不能再把状态当静态模板。
- 当前 `Cluster-A` 的核心职责已经从“补目录和字段边界”切到“明确真实能力、mock 能力、demo payload 与前端 fallback 的边界”。
- 当前最高优先级已经明确为 `D5 ~ D7 live retrieval 稳定性 -> C9 / C11 模式漂移与质量收口 -> G3 / G4 文档同步 -> G2 replay 定稿`。

## 详细子任务

### A1 冻结第一阶段范围与非目标
状态：已完成
目标：确认第一阶段只做最小可运行闭环，不提前扩展到复杂检索平台、多模型路由或重型产品化能力。
产出：统一范围口径。
前置依赖：无。
子子任务清单：
- 整理当前 V1 已承诺能力和明确不做项。
- 对照 `overview/` 与 `rules/` 删除超出第一阶段的扩展目标。
- 输出一版可直接发给各窗口复用的范围说明。
实现备注：顶层 `README.md`、`tasks/README.md` 与各 cluster 任务文件已经统一到“最小闭环已成形，当前重点是把 mock / demo / fallback 推进成真实 reasoning、检索、URL 与回归闭环”的口径；第一阶段不再扩展到复杂多模型路由或重型产品化。
### A2 冻结目录结构与命名边界
状态：已完成
目标：确认 `frontend/`、`backend/`、`contracts/`、`data/`、`tasks/` 的边界与命名规则。
产出：统一目录和命名约定。
前置依赖：A1。
子子任务清单：
- 列出每个顶层目录允许放什么、不允许放什么。
- 约定接口、schema、组件、provider、测试文件的命名方式。
- 形成一份目录边界说明供其他 cluster 遵守。
实现备注：当前顶层目录已经稳定，前端、后端、contracts、evals、data 和 tasks 的边界都已在实际仓库结构中体现。

### A3 维护任务总表与推进顺序
状态：进行中
目标：维护 task 状态、波次顺序和窗口分工，决定当前该启动哪些 cluster。
产出：最新的推进顺序和任务状态。
前置依赖：A1、A2。
子子任务清单：
- 给第一波、第二波、第三波任务排定启动顺序。
- 记录每个 cluster 当前负责人和当前状态。
- 在任务发生阻塞时更新优先级和下一步动作。
实现备注：任务总表已经同步到当前一轮实现状态，并补上了统一的执行记录规则；当前优先级已转向 live retrieval 稳定性、模式漂移与文档同步。后续仍需由主控在每一轮交付后持续回写。
本轮执行任务：
- 把“执行前登记、完成后回写完成方式”的要求正式写入 `tasks/` 体系。
- 给所有 cluster 的全局 prompt 同步这条规则，避免执行窗口漏记。

执行步骤：
- 盘点 `tasks/README.md`、`parallel-execution-playbook.md` 和各 `cluster-*.md` 中缺失的记录要求。
- 在 `tasks/README.md` 中加入执行记录要求和固定模板。
- 在 `parallel-execution-playbook.md` 中加入先登记再执行、完成后回写的流程规则。
- 在各 cluster 的“窗口执行 Prompt（全局）”中同步加入登记与完成记录要求。
- 回读关键文档，确认规则已经落到入口说明和执行 prompt 中。

完成记录：
- 改动文件：`tasks/README.md`、`tasks/parallel-execution-playbook.md`、`tasks/cluster-a-control-tower.md`、`tasks/cluster-b-contract-forge.md`、`tasks/cluster-c-api-foundation.md`、`tasks/cluster-d-retrieval-lab.md`、`tasks/cluster-e-experience-shell.md`、`tasks/cluster-f-quality-gate.md`、`tasks/cluster-g-demo-ops.md`。
- 完成方式：在总入口、执行手册和各 cluster 全局 prompt 三层同时加规则，强制要求执行窗口先把本轮任务写入 task，再开始改动；完成后补完成记录，而不是只改状态词。
- 验证结果：已回读 `tasks/README.md`、`tasks/parallel-execution-playbook.md` 和全部 `cluster-*.md` 的全局 prompt，确认“本轮执行任务 / 执行步骤 / 完成记录”规则已经出现。
- 剩余问题：后续每次实际执行子任务时，仍需由对应窗口在目标子任务下继续补充自己的本轮记录；这条规则不会自动替代具体执行记录。

### A4 审核共享 schema 变更
状态：进行中
目标：审核 `Cluster-B` 的关键字段变化，确保不会影响其他 cluster。
产出：schema 变更决策。
前置依赖：`Cluster-B` 有初版 schema。
子子任务清单：
- 核对 schema 变更是否符合 V1 边界和规则文档。
- 检查该变更是否会影响前端、后端和测试线程。
- 给出通过、驳回或延后执行的决定。
实现备注：`contracts/` 已形成事实上的共享协议基线，前后端当前也已对齐；但变更流程还没有被正式制度化。

### A5 处理跨线程冲突与优先级调整
状态：进行中
目标：当两个 cluster 出现字段、接口或时间安排冲突时给出唯一决策。
产出：冲突处理结论。
前置依赖：进入并行开发后持续存在。
子子任务清单：
- 收集冲突双方的问题描述和受影响文件。
- 判断是字段冲突、职责冲突还是排期冲突。
- 给出唯一执行口径并通知相关 cluster。
实现备注：当前最现实的冲突已经从“字段失控”转成“系统能跑是否等于系统已具备真实能力”的认知不一致；接下来必须由主控明确区分真实 analyze、mock retrieval、demo payload 与前端 safe fallback。
### A6 做里程碑集成验收
状态：进行中
目标：在 mock 闭环完成、真实能力接入完成、演示收口前做 go / no-go 判断。
产出：阶段性是否进入下一波开发的结论。
前置依赖：各波次核心任务完成。
子子任务清单：
- 检查当前里程碑的必要任务是否全部结束。
- 核对测试结果、页面状态和接口可用性。
- 决定继续推进、补缺口还是暂停集成。
实现备注：当前已经具备“最小闭环能跑”的条件，但还没有达到“真实检索、URL、回归测试和最终 README 全部齐备”的下一阶段标准。

### A7 执行最终冻结判断
状态：未完成
目标：在 README、demo case、回归结果齐备后，判断是否进入 V1 演示版冻结。
产出：最终 go / no-go 决策。
前置依赖：前后端、测试、demo 任务基本收口。
子子任务清单：
- 核查演示用例、README、运行步骤是否完整。
- 核查是否仍存在高风险未处理问题。
- 输出最终冻结结论和后续只允许的修改范围。
- 明确只有在真实 reasoning / 检索 / URL / 回归闭环成立后，才允许把系统定义为不依赖 demo / fallback 的 V1 冻结版。
实现备注：当前仍缺真实 live retrieval 通过样本、模式漂移收口、最终口径同步和 replay 定稿，离冻结还有距离。
